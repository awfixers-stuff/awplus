## `bun.sys` — System Calls (`src/sys.zig`)

All return `Maybe(T)` — a tagged union of `.result: T` or `.err: bun.sys.Error`:

```zig

    .result => |fd| fd,
    .err => |err| return .{ .err = err },
};
// Or: const fd = try bun.sys.open(path, bun.O.RDONLY, 0).unwrap();
```

Key functions (all take `bun.FileDescriptor`, not `std.posix.fd_t`):

- `open`, `openat`, `openA` (non-sentinel) → `Maybe(bun.FileDescriptor)`
- `read`, `readAll`, `pread` → `Maybe(usize)`
- `write`, `pwrite`, `writev` → `Maybe(usize)`
- `stat`, `fstat`, `lstat` → `Maybe(bun.Stat)`
- `mkdir`, `unlink`, `rename`, `symlink`, `chmod`, `fchmod`, `fchown` → `Maybe(void)`
- `readlink`, `getFdPath`, `getcwd` → `Maybe` of path slice
- `getFileSize`, `dup`, `sendfile`, `mmap`

Use `bun.O.RDONLY`, `bun.O.WRONLY | bun.O.CREAT | bun.O.TRUNC`, etc. for open flags.

## 5. REMOVAL APPROACH

### Phase 1: Prepare Core Components
1. **Extract resolver** into standalone module with clean interfaces
2. **Stub bundler imports** in files that depend on it but don't need full functionality
3. **Create empty/minimal transpiler** if bundler-specific features are isolated

### Phase 2: Remove Test Runner
1. Delete `src/bun.js/test/` directory
2. Delete `src/cli/test_command.zig` and `src/cli/test/`
3. Delete `src/js/node/test.ts`
4. Update `src/cli.zig` to remove TestCommand
5. Delete test directories (`test/bake/`, `test/js/`, etc.)

### Phase 3: Remove Bundler
1. Delete `src/bundler/` directory
2. Delete `src/cli/build_command.zig`
3. Delete `src/bun.js/api/JSBundler.zig`
4. Update `src/cli.zig` to remove BuildCommand
5. Update `src/bun.zig` to remove bundler exports
6. Stub/remove bundler-dependent code paths in:
   - `src/router.zig`
   - `src/options.zig`
   - `src/OutputFile.zig`

### Phase 4: Clean Up Tests
1. Delete `test/bundler/` directory
2. Delete `test/cli/` (or audit for non-test-runner CLI tests)
3. Delete `test/integration/` (likely has bundler/test deps)

## 6. PACKAGE MANAGER STATUS

The package manager (`src/install/`) has **minimal bundler dependencies**:
- Only imports `src/resolver/dir_info.zig`
- Uses `js_printer` for JSON output
- Does NOT import from `src/bundler/`

**Package manager should survive bundler removal with minimal changes.**
